AetherGeo User Manual

Getting Started

Welcome to the AetherGeo User Manual. This document provides an overview of the functions available in AetherGeo.

To start using Aether Geo the most basic function is to import an image, currently the following format are supported:

  • TIF/TIFF/GeoTIFF
  • HDR in conjunction with DAT files
  • HE5 files

If there seems to be a problem with the visualization, try running the Normalization function under the Preprocessing class, saving it and importing it.

After impoting an image, you can rigth click on top of the layer to check the associated metadata.

Another option is to import spectral libraries with function "Import Spectral Library" in the right side of the screen. Currently the following formats are supported:

  • HDR in conjunction with SLI spectral library
  • TXT spectral library

To continue with the analysis, it is recommended to use the preprocessing functions:

  • Normalization: Performs pixel based normalization to fit the [0,1] interval
  • Attribute Wavelengths: Attributes the correct Wavelengths if they are muissing in the metadata

From this point multiple option can be used under the following groups:

  • 2D plot: Opens a plot and enables the user to save a binary image composed based on manual selection of correlation between data associated with a given image.
  • Dimensionality reduction: Unsupervised techniques that aim to reduce dimensionality while maintaining most relevante information.
  • Endmember extraction: Functions that help the user to identify possible endmembers in the image
  • Spectral Unmixing: Functions that aim to identify the presence of a given material based on the similarity between each pixel spectrum and a input spectrum.
  • Clustering: Unsupervised techniques that perform classification based on spectral data.
  • Spectral libraries: Function that enable the user to import, visualize and compare spectral libraries and respective spectra data.
  • Others: Other relevante function that currently dont fit in a specific category.

For more information on each function, please refer to the specific sections below.

Function Overview

Dimensionality Reduction

Dimensionality reduction techniques are used to reduce the number of features in a dataset while preserving as much information as possible. This is particularly useful in hyperspectral imaging where the number of spectral bands can be very large.

The dimensionality reduction techniques included are the following:

  • Principal Component Analysis (PCA)
  • Independent Component Analysis (ICA)
  • Non-negative Matrix Factorization (NMF)

Each of these techniques has its own strengths and weaknesses, and the choice of which to use will depend on the specific characteristics of the data and the goals of the analysis.

For example, PCA is a linear technique that is effective for data that is approximately Gaussian distributed, while ICA is a non-linear technique that can be used to separate mixed signals. NMF is particularly useful for data that is non-negative (the case for spectral data) and can be used to extract parts-based representations.

To use these functions, select the desired input image, an optional mask, and the number of final components desired. After performing any function a graph describing the results is shown, after this the results can be saved as a new image.

Endmember extraction

Endmember extraction techniques are used to identify the purest spectral signatures in a hyperspectral image. These signatures can then be used to identify the presence of specific materials in the image.

It is recommended to start by performing Pixel Purity Index (PPI) either in the full image, or a mask already highlighting desired areas for regional studies.

This step aim to perform spatial dimensionality reduction aiming to ease the calculation required for the next step.

The "Generate Point Cloud" function creates a 3D visualization of the data.

It is assumed that the endmembers for a given dataset are represented as the boarders of the cloud.

The projection is based on a UMAP embedding of the data. Two images can be used as input, the main image is need as it provides the metadata information. The "reduction result" can be used to have the embedding be based on results from dimensionality reduction, but is not mandatory since UMAP already perform dimensionality reduction on the data.

It is recommended to use results from the PPI function as a mask, to reduce the amount of pixels, reducing computational load, and making identifying extreme ends of clouds easier.

After this save the results from any selection done in the point cloud as a Spectral Library.

Spectral Unmixing

Spectral unmixing techniques are used to identify the presence of specific materials in a hyperspectral image by comparing the spectral signatures of the materials to the spectral signatures of the pixels in the image.

These techniques are particularly useful for identifying materials that are present in small quantities or that are mixed with other materials.

Currently only Spectral Angle Mapper is available.

To use this function, select the desired input image, an optional mask, and an endmember from a spectral library to be used. After performing a new image can be saved where to each pixel is associated a value based on the angle distance between each pixel spectrum and endmember spectrum.

It is recommended to use 2D plot function to inspect results further. In this way the user can save a binary mask where selection was made based on setting a threshold to the previous results.

Clustering

Clustering techniques are used to group similar pixels in a hyperspectral image based on their spectral signatures. This is useful for identifying different materials or land cover types in the image.

Currently the following clustering techniques are available:

  • K-Means Clustering
  • OPTICS Clustering

To use these functions, select the desired input image, an optional mask, and respective necessary parameters. After performing any function a graph describing the results is shown, after this the results can be saved as a new image.

The "Mean Spectra From Cluster" can be used to save a spectral library composed of a mean spectrum for each new cluster generated.

Spectral Libraries

Spectral libraries are collections of spectral signatures that can be used to identify specific materials in a hyperspectral image. AetherGeo allows the user to import, visualize, and compare spectral libraries and respective spectra data.

The "Spectra Analyst" function enables to visualize the spectra associated with a given spectral library. The user can also select two distinct libraries to be visualized simultaneously. SAM can be performed to compare two spectra form either the same or two distinct libraries.

Others

Other relevant functions that currently don't fit in a specific category include:

  • Band Ratios: Generates a image composed of a single layer from the results of an equation applied to data associated with bands from an image.
